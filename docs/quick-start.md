# 快速开始指南

欢迎使用**雪花写作法 Claude Code Subagents**！这份指南将帮助你快速上手。

## 5分钟快速体验

### 1. 安装 Subagents

**方法A：项目级安装（推荐）**
```bash
# 进入你的小说项目目录
cd /path/to/your-novel-project

# 创建 .claude 目录（如果不存在）
mkdir -p .claude

# 复制 subagents
cp -r /path/to/snowflake-novel-subagents/.claude/agents .claude/
```

**方法B：全局安装（适用于所有项目）**
```bash
# 复制到用户目录
cp -r /path/to/snowflake-novel-subagents/.claude/agents ~/.claude/
```

### 2. 验证安装

在 Claude Code 中输入：
```
请列出可用的小说创作 subagents
```

你应该看到 15 个 subagents 的列表。

### 3. 创建你的第一个故事概念

```
请 story-concept-designer 帮我设计一个科幻小说的核心概念。
背景：未来世界，人工智能与人类共存。
主题：探讨意识的本质。
```

Claude 会调用 `story-concept-designer` subagent，为你生成3-5个候选概念并评分。

### 4. 扩展为三幕结构

选定概念后：
```
很好！请 three-act-architect 将这个概念扩展为完整的三幕式结构。
```

## 完整工作流程

### 阶段一：故事框架（1-2周）

#### 步骤1：核心概念
```
请 story-concept-designer 帮我提炼故事核心。
我的创意是：[描述你的初始想法]
```

**产出：** 一句话梗概（30字内）

#### 步骤2：三幕结构
```
请 three-act-architect 将核心概念扩展为三幕式结构。
```

**产出：** 五句话的故事框架

#### 步骤3：人物故事线
```
请 character-storyline-weaver 为主要角色设计故事线。
主要角色有：[列出3-8个角色]
```

**产出：** 每个角色的动机、目标、冲突、感悟

#### 步骤4：一页纸概要
```
请 synopsis-writer 将前三步整合为专业的一页纸概要。
```

**产出：** 800-1000字的故事概要（可投稿）

---

### 阶段二：人物深化（1-2周）

#### 步骤5：人物档案
```
请 character-profiler 为主角创建详细档案。
```

**产出：** 主角一页档案，配角半页档案

#### 步骤6：大纲扩展
```
请 outline-expander 将一页纸概要扩展为四页完整大纲。
```

**产出：** 详细的章节设计

#### 步骤7：人物宝典
```
请 character-bible-creator 建立完整的人物百科。
```

**产出：** 可查询的人物数据库

---

### 阶段三：场景设计（2-3周）

#### 步骤8：场景规划
```
请 scene-planner 规划全书的场景列表。
预计章节数：[X章]
```

**产出：** 完整的场景信息表

#### 步骤9：场景设计
```
请 scene-designer 详细设计第1章第1场景。
```

**产出：** 每个场景的详细蓝图（1000+字）

#### 步骤10：正文撰写
```
请 manuscript-writer 基于场景设计开始创作正文。
场景编号：S001
```

**产出：** 正文初稿

## 使用技巧

### 技巧1：批量处理

可以一次性请求多个 subagent：

```
请按顺序执行：
1. story-concept-designer 设计核心概念
2. three-act-architect 扩展为三幕结构
3. synopsis-writer 生成一页纸概要

主题是：[你的主题]
```

### 技巧2：迭代优化

发现问题时可以返回前面的步骤：

```
我发现主角的动机不够强烈，请 character-storyline-weaver
重新设计主角的故事线，强化其内在驱动力。
```

### 技巧3：混合使用

正文创作时搭配辅助工具：

```
我刚写完第3章，请：
1. consistency-checker 检查一致性问题
2. dialogue-master 优化对话
3. pacing-controller 评估节奏
```

### 技巧4：保存中间产物

每完成一个步骤，保存结果到文件：

```
请将刚才生成的人物档案保存到 characters/protagonist.md
```

## 不同类型小说的工作流调整

### 科幻小说

**额外步骤：**
1. 步骤3后：调用 `genre-specialist`（类型：科幻）验证设定合理性
2. 步骤6后：补充详细的世界观设定文档
3. 使用 `consistency-checker` 频繁检查技术设定一致性

**示例：**
```
请 genre-specialist 从科幻小说专业角度评估我的世界观设定，
指出可能的硬伤和改进方向。
```

### 言情小说

**额外步骤：**
1. 步骤3：特别关注主角和爱人的故事线交织
2. 步骤9：使用 `dialogue-master` 重点打磨情感对话
3. 使用 `pacing-controller` 控制虐甜节奏

**示例：**
```
请 dialogue-master 帮我设计男女主角的初遇对话，
要求：有张力、有误会、有化学反应。
```

### 悬疑推理

**额外步骤：**
1. 步骤2：确保三幕中有明确的线索布局
2. 步骤8：用场景规划表标注"真线索"和"假线索"
3. 步骤9后：使用 `consistency-checker` 验证推理逻辑

**示例：**
```
请 consistency-checker 检查我的推理链条是否严密，
是否有逻辑漏洞或读者可以提前猜到的破绽。
```

### 奇幻小说

**额外步骤：**
1. 步骤4后：创建魔法系统文档
2. 步骤7：人物宝典中增加"能力值"部分
3. 使用 `genre-specialist` 验证世界观自洽性

**示例：**
```
请 genre-specialist 从奇幻小说角度检查我的魔法系统，
确保有限制、有代价、有成长空间。
```

## 常见问题

### Q1: 必须严格按照1-10步骤顺序吗？

**A:** 不必须，但**强烈建议**：
- 前4步是基础，最好按顺序
- 步骤5-7可以并行或调整顺序
- 步骤8-10可以分批进行（先设计部分场景，试写几章再继续）

### Q2: 每个步骤都需要完成吗？

**A:** 根据需求选择：

**最小可行流程（适合短篇或快速验证）：**
- 步骤1 → 步骤2 → 步骤10

**标准流程（适合中长篇）：**
- 步骤1-4（框架）→ 步骤8-10（场景+创作）

**完整流程（适合长篇或严肃创作）：**
- 全部10步 + 辅助工具

### Q3: Subagent 输出的内容可以修改吗？

**A:** 当然可以！Subagent 是**辅助工具**，不是取代你的创作：
- 可以只采纳部分建议
- 可以要求重新生成
- 可以自己修改输出结果

**示例：**
```
这个结局太悲了，请 three-act-architect 重新设计第五句，
改为悲喜交加的开放式结局。
```

### Q4: 如何保存和管理中间产物？

**建议的目录结构：**
```
your-novel-project/
├── .claude/
│   └── agents/           # subagents
├── planning/
│   ├── 01-concept.md     # 步骤1输出
│   ├── 02-structure.md   # 步骤2输出
│   ├── 03-characters.md  # 步骤3输出
│   └── 04-synopsis.md    # 步骤4输出
├── characters/
│   ├── protagonist.md    # 步骤5输出
│   └── bible.md          # 步骤7输出
├── outline/
│   ├── full-outline.md   # 步骤6输出
│   └── scene-list.md     # 步骤8输出
├── scenes/
│   ├── scene-001.md      # 步骤9输出
│   └── scene-002.md
└── manuscript/
    ├── chapter-01.md     # 步骤10输出
    └── chapter-02.md
```

### Q5: 多久应该使用一次一致性检查？

**建议频率：**
- **步骤4后**：检查概念到大纲的一致性
- **每写5章**：检查人物、情节一致性
- **第二幕中点**：全面检查
- **完稿后**：最终全面审查

### Q6: 如何处理创作过程中的灵感变化？

**灵活应对：**

1. **小调整**：直接修改当前步骤产物
   ```
   请更新人物档案，主角的职业从教师改为记者。
   ```

2. **中等调整**：回到相关步骤重做
   ```
   我想改变第二幕的冲突设置，请 three-act-architect
   重新设计第三、四句话。
   ```

3. **大改动**：考虑是否需要重启
   ```
   核心概念需要大改，请 story-concept-designer 基于新的想法
   重新设计，保留[某些元素]。
   ```
