# 科幻小说完整创作流程示例

本示例展示如何使用雪花写作法 subagents 创作一部科幻小说，从概念到成稿的完整流程。

## 故事背景

类型：赛博朋克科幻惊悚
主题：数字永生与人性尊严

---

## 步骤1：故事核心概念设计

**调用：** `story-concept-designer`

**用户输入：**
```
我想写一个关于数字永生的科幻小说，探讨技术进步后人类对死亡的重新定义。
主角是一个黑客，发现了数字永生技术背后的黑暗秘密。
```

**Subagent 输出：**

### 推荐概念A（最终选定）
"当人类意识可以上传云端时，一个黑客发现死去的人仍在数字世界中受苦。"

**三要素评估：**
- ✅ 有趣（5/5）：数字永生是当下热门议题，设定新颖
- ✅ 突兀（5/5）：永生变成折磨的反转出人意料
- ✅ 合理（4/5）：符合科技发展逻辑和商业伦理困境
- **总分：14/15**

**核心冲突：**技术理想 vs 商业邪恶、个人正义 vs 系统压迫

---

## 步骤2：三幕式结构设计

**调用：** `three-act-architect`

**Subagent 输出：**

### 五句话故事梗概

**第一句（建立）：**
2087年的数字永生时代，黑客林昭日复一日地清理数字墓园的垃圾数据，直到他发现一个已故老人的意识碎片在哀嚎求救。

**第二句（进入冲突）：**
林昭决定调查这个异常现象，潜入数字墓园深处，却不知道所有死者意识都被困在永恒的噩梦循环中。

**第三句（冲突升级）：**
调查越深入，林昭越接近惊人真相：数字永生公司刻意制造痛苦以提取"情感数据"，而他已故的女友也在其中受折磨。

**第四句（黑暗时刻）：**
公司追杀、系统崩溃，林昭唯一的脱身机会是删除所有证据——包括女友的意识备份，他必须在爱与正义间做出选择。

**第五句（高潮解决）：**
林昭选择将真相公之于众，引发全球觉醒，虽永失爱人但解放了亿万数字囚徒，人类重新定义了死亡的尊严。

## 步骤3：人物故事线编织

**调用：** `character-storyline-weaver`

**Subagent 输出：**

### 主要角色清单
1. **林昭** - 主角，黑客
2. **陈墨** - 对手，永恒公司CTO
3. **李雪（数字形态）** - 情感支柱，已故女友
4. **老张** - 盟友，自由意识联盟领袖

### 林昭的故事线

**动机：**
- 表层：调查数字异常现象
- 深层：寻找活着的意义（女友死后的虚无感）
- 潜意识：渴望再次与女友相见
- 根源事件：三年前的车祸夺走女友

**目标：**
- 外在：揭露永恒公司的罪行
- 内在：走出丧失的创伤
- 关系：完成对女友的告别

**冲突：**
- 外部：公司追杀、法律威胁
- 内部：放不下过去 vs 面对现实
- 人际：拯救女友 vs 拯救所有人

**感悟：**
- 开始状态：逃避现实、活在过去
- 结束状态：接受失去、活在当下
- 触发点：女友的数字意识告诉他"放手"
- 付出代价：永久失去女友、入狱五年

#### 正文（初稿）

凌晨两点，数字墓园第47区。

林昭的意识飘浮在虚拟空间的深处，周围是无尽的数据流，如同银河系的星云，美丽而冰冷。这是他的工作场所——一个存储了三千万死者意识的数字陵园。

他已经在这里工作了三年。三年来，每天晚上，他都会进入这个空间，清理那些"垃圾数据"——死者意识产生的异常碎片。公司说这是正常现象，就像人类做梦一样。但林昭知道，这不是梦，这是某种他不想深究的东西。

## 辅助工具使用示例

### 使用 dialogue-master 优化对话

**原始对话：**
```
"你在干什么？"陈墨问。
"我在调查真相。"林昭说。
"真相？你知道真相会让你失去什么吗？"
"我不在乎。"
```

**调用 dialogue-master 后：**
```
"林昭。"陈墨的声音从数据流深处传来，平静得可怕。

我没有回答，手指继续在虚拟键盘上飞舞。

"你觉得你在调查什么？正义？"他轻笑，"还是只是在找一个理由，让自己相信李雪还活着?"

我的手停顿了零点三秒。这在虚拟空间里,是一个世纪那么长。

"你不懂。"我听到自己的声音，嘶哑而陌生。

"我很懂。"陈墨说，"因为十年前，我也像你一样。"
```

### 使用 pacing-controller 调整节奏

**原始节奏：**整个第二幕都是快节奏的追逐和对抗

**Subagent 建议：**
- 第15-17章：放慢节奏，加入林昭与李雪意识对话的温情场景
- 第23章：插入一个安静的回忆场景，展示林昭与李雪的过去
- 第28章：在高潮前加入"最后的平静"，让读者喘息

**效果：**张弛有度，情感层次更丰富

### 使用 consistency-checker 检查一致性

**发现的问题：**
1. 第5章林昭说他不会黑入民用系统，第12章却黑了银行
2. 陈墨在第8章是CTO，第20章变成了CEO
3. 数字墓园第47区在第1章位于上海，第18章位于北京
4. 李雪的死亡时间在第3章是2084年3月，第25章是2084年7月

**修复：**
- 在第12章前增加林昭"打破原则"的心理挣扎
- 统一陈墨的职位为CTO
- 修正地点描述，增加"虚拟空间无物理位置"的说明
- 统一李雪死亡时间为2084年3月12日

### 使用 editor-reviewer 最终审阅

**审阅报告（节选）：**

**A级问题（必须修改）：**
1. 第34章的科学原理解释过于复杂，普通读者难以理解
2. 第28章的情感高潮不够，建议加强林昭与李雪的告别场景

**B级问题（建议修改）：**
1. 前三章节奏偏慢，建议压缩到两章
2. 陈墨的动机不够立体，建议增加其背景故事

**C级问题（可选修改）：**
1. 部分技术描述可以更"硬核"
2. 可以增加一些赛博朋克风格的环境描写

### 迭代循环

雪花写作法不是严格线性的，可以循环迭代：

```
概念(1) → 结构(2) → 人物(3) → 概要(4)
   ↑                                  ↓
   ←――――――― 发现问题，返回修改 ――――――←

人物档案(5) → 大纲(6) → 人物宝典(7)
   ↑                           ↓
   ←――――― 细化发现新设定 ――――←

场景规划(8) → 场景设计(9) → 正文(10)
   ↑                           ↓
   ←――――― 写作中调整 ――――――――←
```

## 经验总结

### 最有价值的步骤

1. **步骤3（人物故事线）**：确保角色立体，避免"工具人"
2. **步骤8（场景规划）**：全局视角，避免结构失衡
3. **辅助工具（一致性检查）**：在长篇创作中极其重要

### 常见陷阱

1. ❌ 跳过前期步骤直接写正文 → 容易写崩
2. ❌ 过度规划导致失去创作激情 → 保持适度灵活性
3. ❌ 忽视一致性检查 → 后期修改成本巨大

### 最佳实践

1. ✅ 每完成一个步骤都休息一天，避免思维僵化
2. ✅ 准备"灵感笔记"记录临时想法
3. ✅ 定期使用 consistency-checker 检查
4. ✅ 找测试读者在步骤4和步骤9后提供反馈
